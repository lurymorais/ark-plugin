## 2. Data Collection

### Default: Data is Sent (Opt-out)

By default, the plugin sends anonymous usage data monthly to help improve the service. You can disable this at any time in the plugin settings.

**Data sent (default, unless disabled):**

- naan: Public NAAN identifier (ex.: ark:16081)
- arks_count: Number of ARKs generated by the journal (ex.: 67)
- plugin_version: ARK Plugin version (ex.: 3.1.0.0)

**NOT collected:**

- No administrator emails
- No IP addresses (only for rate limiting, not stored)
- No user data
- No content data
- No personal identifiable information (PII)

### NAAN is Public Data

NAAN (Name Assigning Authority Number) is:

- A public technical identifier assigned by ARK Alliance
- Publicly queryable, e.g.: https://n2t.net/ark:16081
- Not personal data (identifies an organization, not an individual)

---

## 3. How Data is Used

### Aggregated Statistics

Data collected is used to:

1. Count total ARKs generated across the OJS ecosystem
2. Count unique journals using the plugin
3. Track plugin versions in use (to prioritize support)

Example output (public dashboard):

- total_arks_generated: 567
- unique_journals: 45

### No Individual Identification

- Individual journal data is never published
- No journal names, URLs, or identifying information is exposed
- Only aggregate numbers are published publicly

---

## 4. Data Retention

- Statistics: Permanent (aggregate only)
- Validation logs: 24 months (automatic deletion)
- Rate limit records: 24 hours (automatic deletion)
- Validation tokens: 5 minutes (automatic expiration)
- Consent audit: 5 years (LGPD/GDPR requirement)

### Requesting Deletion

You can request deletion of your journal's statistics data:

1. Disable telemetry: Go to Settings > ARK Plugin > Uncheck "Send anonymous statistics"
2. Contact us: Email m.luryhortencio@gmail.com with your NAAN

Response time: Within 30 days (as required by GDPR/LGPD)

---

## 5. Data Storage and Security

### Storage Location

Data is stored on a secure server in Brazil (revistacarnaubais.com.br).

### Security Measures

- Encryption: SSL/TLS for all data transmission
- Access Control: Only the data controller (Lury Morais) has access
- Rate Limiting: Prevents abuse/brute force
- No API Keys: No permanent credentials stored
- No Tokens: No persistent authentication tokens

### Data Protection

- No personal data: By design, no PII is collected
- Aggregation only: Data is only used for aggregate statistics
- 24-month retention: Automatic deletion prevents indefinite storage

## 6. Your Rights (GDPR/LGPD)

You have the following rights regarding your data:

- Right to Access: Request a copy of your journal's data
- Right to Rectification: Correct any inaccurate data
- Right to Erasure: Request deletion of your journal's data
- Right to Restrict Processing: Disable telemetry in plugin settings
- Right to Data Portability: Request your data in a machine-readable format
- Right to Object: Disable telemetry entirely

### How to Exercise Your Rights

Email m.luryhortencio@gmail.com with:

1. Your NAAN (if known)
2. Your journal URL
3. Specific right you want to exercise

Response time: Within 30 days

## 7. Third-Party Services

### n2t.net (NAAN Validation)

When validating a NAAN, the plugin queries https://n2t.net:

- Purpose: Verify NAAN exists and domain matches
- Data sent: NAAN only (public data)
- Data stored: None

### ARK Alliance

NAANs are registered with ARK Alliance (https://n2t.net/ark:):

- Purpose: Provide persistent identifiers
- Data: NAAN and journal URL (public)
- Privacy: NAANs are inherently public identifiers
